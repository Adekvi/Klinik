Optimasi Fitur perUser
1. Superadmin
- tabel master seluruh pasien untuk import data manual, ketika setiap kolom datanya tidak sama maka data tidak akan bisa teruploud ke dalam tabel
- perlu adanya peringatan ketika data yang diuploud tidak sesuai dengan kolom agar user bisa mengetahui jika ada kesalahan nama kolom

2. Dokter umum dan Dokter Gigi
- Soap pasien untuk update data belum terambil keseluruhan nilai/value yang telah diinput sebelumnya
- Masing-masing kolom untuk penambahan obat didalam soap, jenis obat, anjuran minum, aturan minum ada tabel masternya sehingga data bisa lebih dinamis dan bisa diubah oleh admin/user yg mempunyai akses
- belum mengecek lagi untuk input gambar disoap dokter umum, input data telah sesuai untuk edit juga seharusnya sudah sesuai, perlu dicek lagi
- apakah baiknya ada perubahan untuk aturan mengedit data soap dengan batas waktu, agar ketika data sudah lebih dari hari yang ditentukan tidak bisa diubah lagi?

3. User Apoteker
- menyesuaikan dengan input obat dari hasil soap dokter
- permasalahan yang sering terjadi adalah dimana antara jumlah obat dengan jumlah per obatnya tidak sama, atau lebih banyak untuk jumlah per nama obatnya, sehingga penentuan untuk harga obatnya tidak muncul, karena harga diambil dari data nama obat
- hak akses untuk mengubah master data aturan minum, anjuran minum dan nama obat
- untuk tabel resep perlu ditambahkan untuk keterangan nama obat -> sebelum minum, sesudah minum, dll

4. User Kasir
- tinggal menyesuaikan dengan tabal dari total obat yang telah dicek dari user apoteker
